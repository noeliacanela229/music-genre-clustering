Clustering Music Genres

Noelia Canela & Ethan Ferguson — CS 472 Final Project

What We Did

We wanted to see if a clustering algorithm could figure out music genres on its own just from audio data — no labels, no hints. We used a Spotify dataset with about 32,000 songs and ran 6 different clustering algorithms to compare which one works best.

Dataset

TidyTuesday Spotify dataset — 32,828 songs across 6 genres (edm, rap, pop, r&b, latin, rock). Loads automatically from a URL so no download needed.

Features used: danceability, energy, loudness, speechiness, acousticness, instrumentalness, liveness, valence, tempo

Algorithms

  • K-Means
  • K-Means++
  • K-Medoids
  • DBSCAN
  • GMM (Gaussian Mixture Model)
  • Hierarchical Clustering

Results

Algorithm Silhouette Score Notes
K-Means 0.1384 Best performer
K-Means++ 0.1384 Tied for best
Hierarchical 0.0992 5k sample
K-Medoids 0.0841
GMM -0.0176 Soft assignments didn't fit
DBSCAN N/A 46 clusters, 16,606 noise points

Key Findings

  • K-Means and K-Means++ tied for best at 0.1384
  • DBSCAN completely failed — music genres aren't dense enough for density-based clustering
  • GMM went negative — hard assignments worked better than soft probabilities for this data
  • Cluster 5 had a tempo of 169 BPM — way faster than all other clusters which averaged 110–125
  • Energy and loudness are highly correlated at 0.68 — explains why EDM and rock end up close together

How to Run

pip install -r requirements.txt
python3 music_genre_clustering.py

The dataset downloads automatically. Close each plot window as it pops up so it keeps running.
